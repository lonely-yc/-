# 项目工时计划智能汇总与饱和度评估台设计规格

## 背景

根据《AI 赋能·数智创新实践赛》通知，参赛作品应为浏览器可访问的 BS 端小工具，解决日常工作中的真实问题，主体功能可正常运行，并使用 mock 或脱敏数据演示。

本设计聚焦项目经理和开发/现场人员的日常填报场景：每天需要在 Excel 中填写项目名称、任务模块、详细任务、工作量、开始时间、结束时间、完成进度和备注。当前痛点是项目经理需要手工汇总、判断人员投入是否饱和、识别进度异常、整理日报，重复劳动多且容易漏看异常。

## 候选工具方向

### 1. 项目工时计划智能汇总与饱和度评估台

员工仍按 Excel 模板填写每日计划，项目经理上传后，系统自动完成字段校验、工时汇总、人员饱和度估算、完成时效星级评定、风险提示和日报生成。

这是主推方向，贴近日常工作，演示路径清晰，完成度容易做高。

### 2. 项目日报/周报自动生成器

以工时计划 Excel 为输入，自动生成项目日报或周报，内容包括项目投入、人员工作内容、进度摘要、风险问题和后续计划。

适合作为主推方向的增强模块，不建议单独作为首选作品。

### 3. 开发计划饱和度与完成质量评估器

重点根据人员投入项目数量、工作量、工作内容复杂度、完成进度和完成时间估算工作饱和度，并输出完成时效星级。

管理价值明显，但需要规范的日期和人员字段支持。

### 4. 项目问题闭环轻看板

用于登记项目问题、责任人、优先级、截止时间、状态流转和逾期提醒，帮助项目经理追踪问题闭环。

页面展示直观，但与当前 Excel 工时填报场景关联较弱，可作为后续扩展方向。

## 选定方案

最终选定方案为：项目工时计划智能汇总与饱和度评估台。

产品定位不是替代现有 Excel，而是给项目经理一个 Excel 增强工作台。员工继续使用熟悉的表格填报，系统负责把表格数据转成可校验、可汇总、可评估饱和度、可生成日报的管理视图。

## 目标用户

### 开发/现场人员

关注自己当天投入哪个项目、做了哪些任务、计划时间、工时和进度是否填写完整。

### 项目经理

关注每个项目投入多少工时、哪些人员工作饱和、哪些任务进度异常、哪些任务完成较及时，以及日报/周报能否快速生成。

## Excel 数据字段

系统以现有 Excel 字段为基础，并扩展少量用于分析的字段。

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 序号 | 数字 | 否 | 用于行号展示和错误定位 |
| 日期 | 日期 | 是 | 用于日报、周报和按天汇总 |
| 姓名 | 文本 | 是 | 用于人员饱和度和投入分析 |
| 项目名称 | 文本 | 是 | 用于项目维度汇总 |
| 任务模块 | 文本 | 是 | 用于模块维度分析 |
| 详细任务 | 文本 | 是 | 用于任务明细和日报生成 |
| 工作类型 | 文本 | 否 | 可选值包括开发、测试、联调、会议、现场支持、文档 |
| 工作量(小时) | 数字 | 是 | 用于工时统计和超负荷判断 |
| 开始时间 | 时间 | 是 | 用于计划跨度和饱和度估算 |
| 结束时间 | 时间 | 是 | 用于计划跨度和饱和度估算 |
| 完成进度 | 文本或百分比 | 是 | 用于进度状态分析 |
| 实际完成时间 | 时间或日期时间 | 否 | 用于完成时效星级评定；如果没有该列，初版使用结束时间和完成进度估算 |
| 备注 | 文本 | 否 | 用于记录风险、阻塞和补充说明 |

## 核心校验规则

1. 项目名称、任务模块、详细任务、工作量(小时)、开始时间、结束时间、日期、姓名不能为空。
2. 工作量(小时)必须大于 0。
3. 结束时间必须晚于开始时间。
4. 同一姓名、同一日期下，不同项目的时间段允许重叠，不直接判定为错误；系统仅将其作为并行投入和饱和度计算依据。
5. 同一姓名、同一日期总工作量超过 8 小时标黄，超过 10 小时标红。
6. 详细任务少于 6 个字时标黄，提示任务描述不够清楚。
7. 完成进度低于 100% 且备注为空时标黄，提示补充未完成原因或后续计划。
8. 工作量与开始/结束时间跨度明显不一致时标黄。

## 工作饱和度模型

系统按“姓名 + 日期”计算人员工作饱和度。初版采用规则评分，不依赖真实 AI 模型，便于解释和演示。

饱和度输入包括：

- 当日总工作量。
- 当日投入项目数量。
- 当日任务数量。
- 详细任务字数和关键词复杂度。
- 完成进度。
- 是否存在备注中的阻塞、等待、延期、联调等关键词。

初版饱和度分为 4 档：

| 饱和度 | 规则示例 | 展示方式 |
| --- | --- | --- |
| 空闲 | 总工时小于 4 小时，任务少于 2 条 | 绿色 |
| 正常 | 总工时 4-8 小时，任务和项目数量适中 | 蓝色 |
| 饱和 | 总工时 8-10 小时，或同时投入 3 个及以上项目 | 黄色 |
| 过载 | 总工时超过 10 小时，或高工时且进度偏低 | 红色 |

同一个人员、同一个时间段、不同项目可以重叠。重叠不作为冲突错误，而是在人员饱和度页中展示为“并行投入”，并参与饱和度估算。

## 完成时效星级评定

系统根据完成进度和完成时间对任务进行星级评定，用于帮助项目经理判断任务完成效率。

初版星级规则：

| 星级 | 规则示例 | 含义 |
| --- | --- | --- |
| 5 星 | 完成进度 100%，实际完成时间不晚于计划结束时间 | 准时完成 |
| 4 星 | 完成进度 100%，实际完成时间晚于计划结束时间但仍在当天完成 | 轻微延期 |
| 3 星 | 完成进度 80%-99%，且备注说明清楚 | 基本可控 |
| 2 星 | 完成进度 50%-79%，或备注缺少延期原因 | 需要关注 |
| 1 星 | 完成进度低于 50%，或存在明显延期且无说明 | 高风险 |

如果 Excel 没有“实际完成时间”字段，初版按完成进度和计划结束时间估算星级：完成进度为 100% 视为按计划完成；未完成任务按进度和备注完整度评定 1-3 星。

## 页面设计

### 1. 首页看板

首页用于展示导入数据后的整体情况，适合作为比赛演示首屏。

核心指标包括：

- 今日总工时
- 参与人员数
- 涉及项目数
- 进行中任务数
- 异常记录数
- 高风险提示数

下方展示两个区域：

- 项目投入排行：项目名称、累计工时、任务数、平均完成进度。
- 饱和度与风险提示：人员、项目、饱和度、星级、提示类型。

### 2. Excel 导入页

导入页采用三步流程：

1. 上传 Excel。
2. 预览识别字段。
3. 查看校验结果。

导入后展示表格预览，并给出通过记录、警告记录和错误记录数量。错误需要定位到具体行，例如“第 6 行：结束时间早于开始时间”。

如果字段名称与模板不完全一致，初版可以先提示用户调整模板；后续版本再支持字段映射。

### 3. 工时汇总页

工时汇总页提供日期范围、项目名称、姓名、完成进度、饱和度和星级筛选。

中间使用 Tab 切换：

- 按项目汇总
- 按人员汇总
- 按任务模块汇总

项目经理点击某个项目后，可查看该项目下的任务模块、参与人员、详细任务和累计工时。

### 4. 人员饱和度页

人员饱和度页用于展示每个人每日总工时、项目投入、任务数量、并行投入和饱和度评估。

左侧为人员列表，显示每个人今日总工时、投入项目数、任务数和饱和度等级。右侧为 09:00-18:00 时间轴，展示选中人员的任务安排。

如果同一人员同一天存在不同项目时间段重叠，重叠区域展示为“并行投入”，不直接标红。系统结合总工时、项目数量、任务数量、工作内容和进度计算饱和度。

### 5. 饱和度与风险提示页

饱和度与风险提示页按类型筛选：

- 工作过载
- 多项目并行
- 单日超工时
- 进度异常
- 备注缺失
- 任务描述过短
- 工时与时间跨度不一致
- 星级偏低

每条提示展示等级、姓名、项目名称、饱和度、星级、问题说明和建议处理方式。

### 6. 日报/周报生成页

用户选择日期范围和项目后，系统生成结构化日报或周报。

日报内容包括：

- 今日投入
- 主要工作
- 风险问题
- 饱和度与星级摘要
- 后续计划

初版支持复制文本和导出为 Markdown 或文本文件。后续可扩展 Word 或 Excel 导出。

## 推荐前端布局

整体采用企业后台工作台风格。

- 顶部：系统名称、当前导入批次、日期范围、导出按钮。
- 左侧导航：首页看板、Excel 导入、工时汇总、人员饱和度、风险提示、日报生成。
- 主区域：表格、指标卡、图表、风险列表、生成结果。
- 右上角：演示数据、清空数据、导出报告。

视觉风格应清爽克制，使用白底、浅灰分区、蓝色主操作按钮，并用红、黄、绿表达风险等级。

## 数据流

1. 用户上传 Excel 文件。
2. 前端解析 Excel 数据。
3. 系统转换为统一的任务记录结构。
4. 校验模块生成错误和警告。
5. 汇总模块生成项目、人员、日期和任务模块统计。
6. 饱和度模块生成人员饱和度和任务星级。
7. 风险提示模块生成过载、进度异常、备注缺失等提示清单。
8. 报告模块根据筛选条件生成日报或周报文本。
9. 用户查看、复制或导出结果。

## 错误处理

- 文件格式错误：提示仅支持 xlsx 或 xls。
- 缺少必填字段：提示缺失字段名称，并建议下载或参考模板。
- 单元格格式异常：定位到具体行和字段。
- 存在错误记录：允许用户查看问题，但不纳入正式汇总。
- 存在警告记录：允许纳入汇总，同时在风险提示页展示。
- 无有效数据：显示空状态和重新上传入口。

## 比赛演示脚本

建议 3 分钟演示：

1. 展示原始 Excel，说明每天靠人工汇总和检查效率低。
2. 上传 mock Excel，系统自动识别字段并校验。
3. 打开首页看板，展示项目数、人员数、总工时、饱和人员数和低星任务数。
4. 打开人员饱和度页，演示同一人员多项目并行时如何估算工作饱和度。
5. 打开风险提示页，展示系统给出的过载、进度异常和星级偏低提示。
6. 打开日报生成页，生成一段可复制的项目日报。
7. 总结价值：减少人工汇总、减少漏填错填、快速判断人员是否饱和和任务完成质量。

## 初版范围

初版必须完成：

- mock Excel 数据导入或内置演示数据加载。
- 表格预览。
- 基础字段校验。
- 项目、人员、任务模块维度汇总。
- 工作饱和度估算。
- 完成时效星级评定。
- 风险提示列表展示。
- 日报文本生成。
- 浏览器可访问并可一键运行。

初版不做：

- 登录和权限。
- 多用户协作。
- 后端数据库。
- 对接真实内部系统。
- 使用真实客户数据或未脱敏内部数据。

## 技术边界

推荐实现为前端优先的轻量 BS 工具。可使用 mock 数据和浏览器本地解析 Excel，降低部署复杂度。

如果使用 React 或 Vue，可结合表格组件、图表组件和 xlsx 解析库实现。若时间紧，也可以使用原生 HTML、CSS、JavaScript 实现单页工具。

数据可以存储在浏览器内存或 localStorage 中，满足比赛演示即可。

## 测试与验收

验收标准：

1. 能打开浏览器页面。
2. 能加载 mock 数据或上传示例 Excel。
3. 能看到字段校验结果。
4. 能按项目和人员查看工时汇总。
5. 能计算人员工作饱和度，并识别至少一种超工时或进度异常提示。
6. 能生成日报文本。
7. 页面主要流程无明显报错。

建议测试数据至少包含：

- 正常任务记录。
- 缺少必填字段的记录。
- 结束时间早于开始时间的记录。
- 同一人员同一时间段投入不同项目的记录。
- 单日工时超过 8 小时的记录。
- 完成进度低于 100% 且备注为空的记录。
- 完成进度为 100% 且实际完成时间早于或晚于计划结束时间的记录。
