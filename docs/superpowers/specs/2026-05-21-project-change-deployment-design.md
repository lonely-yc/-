# 项目需求变更与发包部署追踪台设计规格

## 背景

根据《AI 赋能·数智创新实践赛》通知，参赛作品应为浏览器可访问的 BS 端小工具，解决日常工作中的真实问题，主体功能可正常运行，并使用 mock 或脱敏数据演示。

本设计聚焦项目交付后期的需求变更和发包部署管理。实际项目中，客户需求、前后端改动、数据库或配置调整、实施发包、中间件 IP 和端口等信息常分散在群聊、Excel、代码提交和实施记录中，项目经理复盘时难以形成完整链路。

## 产品定位

项目需求变更与发包部署追踪台是一个项目经理视角的交付过程追踪工具。

初版不做账号权限，不做真实审批，不对接 Git 或发包系统。系统重点把项目台账、需求变更、开发改动、发包部署和中间件配置串起来，形成可查询、可追踪、可统计的交付链路。

## 目标用户

### 项目经理

关注项目后期需求是否频繁变更、每次变更改了什么、是否已经发包、发包是否成功、现场环境配置是否有记录。

### 开发经理

关注每条需求变更涉及哪些前端、后端、数据库、配置或设计改动，以及这些改动是否已关联到版本或发包批次。

### 实施人员

关注每次发包部署的环境、包版本、中间件、现场 IP、端口、配置说明和部署结果。

初版统一以项目经理工作台形式呈现，不做真实登录和权限隔离。

## 核心目标

1. 让项目经理能从一个页面追踪“需求变更 → 开发改动 → 发包部署 → 中间件配置 → 部署结果”。
2. 让后期需求变更可标记、可查询、可统计。
3. 让每次发包能说明关联了哪些需求、为什么发包、改动了什么配置。
4. 让项目复盘时能看到变更次数、发包频率、需求到发包耗时、后期变更占比。
5. 保留 Excel 台账作为现实入口，初版支持 mock 数据和 Excel 导入入口。

## 数据来源

采用“Excel 导入 + 页面维护”的方式。

初版内置 mock 数据，提供 Excel 上传入口。页面可以新增和编辑记录。字段映射、复杂 Excel 模板兼容和真实系统对接放到后续版本。

## 页面设计

### 1. 首页看板

首页用于展示项目交付整体情况。

核心指标：

- 项目数
- 需求变更数
- 开发改动数
- 发包次数
- 后期变更数
- 部署异常数

下方展示：

- 高频变更项目排行
- 近期发包记录
- 待跟进变更

### 2. Excel 导入

支持导入或加载四类台账：

- 项目台账
- 需求变更台账
- 开发改动台账
- 发包部署台账

初版提供 mock 数据加载和 Excel 上传入口。导入后展示识别记录数、缺失字段、无法关联的变更编号或项目编号。

### 3. 需求变更

展示需求变更列表，支持按项目、变更类型、是否后期变更、优先级、状态筛选。

每条记录展示提出时间、提出人、变更内容、变更原因、影响范围和当前状态。

初版支持页面新增和编辑记录。

### 4. 开发改动

按变更编号查看关联的开发改动。

重点展示：

- 前端改动
- 后端改动
- 数据库改动
- 配置改动
- 设计或实施配合项

每条改动记录展示负责人、完成时间、关联版本或分支。

### 5. 发包部署

展示每次发包部署记录。

重点展示：

- 发包环境
- 发包原因
- 关联变更编号
- 前后端包版本
- 中间件名称
- 现场 IP 和端口
- 配置文件说明
- 实施人员
- 部署结果

初版不上传真实配置文件，只记录配置文件说明和关键参数。

### 6. 项目追踪链

这是作品核心亮点页面。

用户选择项目或变更编号后，系统展示：

需求变更 → 开发改动 → 发包部署 → 中间件/配置 → 部署结果

每个节点展示时间、负责人、状态和关键说明。无法关联的记录需要标记为“未关联”，便于发现台账缺口。

### 7. 统计分析

展示项目交付过程统计。

统计指标：

- 每项目变更次数
- 每项目发包次数
- 发包频率
- 需求到发包平均耗时
- 后期变更占比
- 变更类型分布
- 部署成功/异常占比

初版采用 mock 数据和规则计算，不引入复杂图表库也可以完成，优先保证表格和指标准确。

## 核心计算规则

### 后期变更占比

后期变更占比 = 是否后期变更为是的需求变更数量 / 需求变更总数量。

### 发包频率

发包频率 = 统计周期内发包次数。初版按项目统计总发包次数，并展示最近一次发包时间。

### 需求到发包耗时

对每条已经关联发包记录的需求变更，计算首次发包时间 - 提出时间。按项目取平均值。

### 变更改动率

变更改动率 = 已关联开发改动的需求变更数量 / 需求变更总数量。

### 部署异常数

部署异常数 = 部署结果为失败、回滚或待验证的发包记录数量。

## 错误与异常处理

- 缺少必填字段：在导入页显示字段名称和记录行号。
- 项目编号不存在：标记为“未关联项目”。
- 关联变更编号不存在：标记为“未关联变更”。
- 一个发包记录关联多个变更编号：拆分展示关联关系。
- 日期格式无法识别：提示用户修正台账。
- 无数据：各页面显示空状态和加载演示数据入口。

## 比赛演示脚本

建议 3 到 5 分钟演示：

1. 打开首页看板，说明当前管理 3 个项目、12 条需求变更、8 次发包。
2. 进入 Excel 导入页，加载 mock Excel 台账。
3. 进入需求变更页，展示某客户在项目后期提出紧急调整。
4. 进入开发改动页，查看该变更关联的前端页面、后端接口、数据库或配置改动。
5. 进入发包部署页，展示现场发包记录、前后端版本、中间件、IP、端口和配置说明。
6. 进入项目追踪链页，一屏串起“需求 → 开发 → 发包 → 配置 → 结果”。
7. 进入统计分析页，展示变更率、发包频率、需求到发包耗时和部署异常。
8. 总结价值：交付过程可追溯、后期需求可统计、发包原因和配置变更可复盘。

## 初版范围

初版必须完成：

- 单页 BS 应用。
- mock 数据。
- Excel 导入入口。
- 四类台账列表。
- 新增和编辑表单。
- 项目追踪链。
- 统计看板。
- 浏览器可访问并可一键运行。

初版不做：

- 登录权限。
- 多角色审批。
- Git 对接。
- 发包系统对接。
- 真实配置文件解析。
- 真实客户数据或未脱敏内部数据。

## 技术边界

推荐继续使用当前 React、TypeScript、Vite 前端项目。数据保存在浏览器内存中，满足比赛演示即可。

Excel 解析可复用现有 xlsx 依赖。导入时先支持结构化台账数据，复杂字段映射放到后续版本。

页面以项目经理工作台为主，不做路由系统也可以完成，使用左侧导航切换模块即可。

## 测试与验收

验收标准：

1. 能打开浏览器页面。
2. 能加载 mock 数据。
3. 能展示项目台账、需求变更、开发改动和发包部署台账。
4. 能新增和编辑至少一种台账记录。
5. 能按项目或变更编号展示完整追踪链。
6. 能计算变更次数、发包次数、后期变更占比、部署异常数。
7. 页面主要流程无明显报错。

建议测试数据至少包含：

- 一个正常项目。
- 一个后期紧急变更。
- 一个变更关联多条开发改动。
- 一个发包关联多个变更编号。
- 一个现场环境部署记录，包含中间件、IP、端口和配置说明。
- 一个部署失败或待验证记录。
- 一个无法关联的变更编号，用于展示台账缺口。
